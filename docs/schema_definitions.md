# Axiom Schema Definitions v0.1

This document outlines the standard JSON schema specification for Axiom entities. All schemas are designed to be human-readable, machine-validatable, and portable.

## 1. Global Schema Rules
All Axiom objects MUST contain the following fields:
- `id` (string): Unique identifier for the object. Dot-notation is required for namespaces (e.g., `namespace.type.name` like `sales.email.outreach`).
- `type` (string): The entity type (`template`, `prompt`, `skill`, `usecase`).
- `version` (string): SemVer compliant version string (e.g., `1.0.0`).
- `inputs` (object): Defined as a JSON Schema typing block to ensure strict validation (e.g., `{"text": {"type": "string"}}`).
- `metadata` (object): Key-value pairs for descriptive data.

Objects MAY contain:
- `config` (object): Execution configuration (e.g., `timeout`, `retries`) or model configuration (e.g., `temperature`, `max_tokens`).
- `capability` (array of strings): Tags for capability-based indexing (e.g., `["summarization", "reasoning"]`).

## 2. Template Schema
The lowest-level reusable primitive. Defines the structured message array with roles.

```json
{
  "$schema": "https://axiom.dev/schema/v0.1/template.json",
  "id": "summarize.base",
  "type": "template",
  "version": "1.0.0",
  "inputs": {
    "text": { "type": "string" },
    "style": { "type": "string", "enum": ["bullet points", "paragraph"] }
  },
  "messages": [
    {
      "role": "system",
      "content": "You are a helpful summarization assistant."
    },
    {
      "role": "user",
      "content": "Please summarize the following text in {{style}}:\n{{text}}"
    }
  ],
  "capability": ["summarization"],
  "metadata": {
    "author": "prompt-engineering-team",
    "description": "A reusable structured conversational template."
  }
}
```

## 3. Prompt Schema
An executable instance of a Template. It binds specific config or overrides.

```json
{
  "$schema": "https://axiom.dev/schema/v0.1/prompt.json",
  "id": "summarize.basic",
  "type": "prompt",
  "version": "1.0.0",
  "extends": "summarize.base",
  "inputs": {
    "text": { "type": "string" },
    "style": { "type": "string", "default": "paragraph" }
  },
  "config": {
    "temperature": 0.2,
    "max_tokens": 1000
  },
  "metadata": {
    "description": "A strict configured prompt derived from the base summarize template."
  }
}
```

## 4. Skill Schema
A Skill aggregates prompts or defines a specific execution strategy.

```json
{
  "$schema": "https://axiom.dev/schema/v0.1/skill.json",
  "id": "rag.document_qa",
  "type": "skill",
  "version": "1.0.0",
  "inputs": {
    "query": { "type": "string" },
    "document_id": { "type": "string" }
  },
  "steps": [
    "retrieve_chunks",
    "rank_chunks",
    "generate_answer"
  ],
  "strategy": "pipeline",
  "config": {
    "timeout": 30000,
    "retries": 3
  },
  "metadata": {
    "description": "Standard Document Retrieval-Augmented Generation pipeline."
  }
}
```

## 5. UseCase Schema
A specialized assembly of Skills targeted at solving a holistic user objective.

```json
{
  "$schema": "https://axiom.dev/schema/v0.1/usecase.json",
  "id": "customer_support.inquiry_resolution",
  "type": "usecase",
  "version": "1.0.0",
  "inputs": {
    "customer_message": { "type": "string" },
    "customer_profile": { "type": "object" }
  },
  "skills": [
    "intent_classification",
    "rag.document_qa",
    "response_formatter"
  ],
  "metadata": {
    "department": "support",
    "criticality": "high"
  }
}
```

## 6. ExecutionPlan Schema
The deterministic graph structure generated by the Axiom Runtime and handed to Framework Adapters.

```json
{
  "$schema": "https://axiom.dev/schema/v0.1/execution_plan.json",
  "id": "plan.customer_support.1234",
  "entrypoint": "customer_support.inquiry_resolution",
  "nodes": {
    "step_1": {
      "type": "skill",
      "ref": "rag.document_qa@1.0.0",
      "resolved_prompts": ["summarize.basic@1.0.0"]
    }
  },
  "edges": [
    { "from": "start", "to": "step_1" }
  ],
  "resolved_inputs": {
    "customer_message": "My order never arrived."
  }
}
```

## 7. Storage Recommendations
Files should be organized functionally or by entity type.
```text
/axiom_registry
  /templates
    summarize.base.json
  /prompts
    summarize.basic.json
    extract.entities.json
  /skills
    rag.document_qa.json
  /usecases
    customer_support.json
```
